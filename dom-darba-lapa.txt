1. uzdevums – DOM jēdziens (teorija)

1.1. Kas ir DOM?
Uzraksti skaidrojumu saviem vārdiem (1–2 teikumi):
Tas ir saites struktūras attēlojums kuru var nolasīt JavaScript. Tas ļauj dinamiski pārvaldīt kodu

1.2. Kāpēc DOM ir nepieciešams dinamiskos tīmekļa produktos?
Tas ir svarīgs, lai mainītu lapas saturu, stilu, struktūru bez pārlādes, bez tas saite nereaģetu uz lietotāja darbībām
2. uzdevums – DOM elementu atlase

2.1. Nosauc vismaz 3 DOM elementu atlases metodes:

1. document.getElementById()
2. document.getElementsByClassName()
3. document.uerySelector()

2.2. Kuru atlases metodi Tu šodien izmantoji visbiežāk? Kāpēc?
document.getElementById(), jo tā pievieno elementam unikālo ID. Viegli izmantojama

3. uzdevums – Esoša HTML elementa maiņa ar JavaScript


Ievieto koda fragmentu, kurā:

1. tiek atlasīts HTML elements;
2. tiek mainīts tā saturs;
3. tiek mainīta vismaz viena stila īpašība.

// Ievieto savu JavaScript kodu šeit
const teksts = document.getElementById("teksts");
teksts.textContent = "Mainīts ar JavaScript";
teksts.style.color = "green";

3.1. Apraksti, kas notiek lapā, kad šis kods tiek izpildīts:
Kad kods darbojas, JavaScript atrod elementu ar ID "teksts" un nomainā tas saturu uz jaunu tekstu un paradāra tekstu zāļā krāsā.
4. uzdevums – Dinamiska HTML elementa izveide

Ievieto koda fragmentu, kurā:

1. ar JavaScript tiek izveidots jauns HTML elements;
2. tas tiek pievienots lapai.

// Ievieto savu JavaScript kodu šeit
const jaunsParagrafs = document.createElement("p");
jaunsParagrafs.textContent = "Šis elements tika izveidots dinamiski";
document.body.appendChild(jaunsParagrafs);

5. uzdevums – HTML elementa noņemšana

Ievieto koda fragmentu, kurā iepriekš izveidotais elements tiek noņemts no lapas.

// Ievieto savu JavaScript kodu šeit
jaunsParagrafs.remove();
